## Оцінювання ефективності алгоритмів

Для порівняння ефективності жадібного алгоритму та алгоритму динамічного
програмування було проведено вимірювання часу їх виконання для різних сум.

Сума | Greedy (сек) | DP (сек)
-----------------------------------
  100 | 0.000010 | 0.000526
 1000 | 0.000007 | 0.006624
 5000 | 0.000008 | 0.037245
10000 | 0.000007 | 0.067082

Жадібний алгоритм показав стабільний та дуже швидкий час виконання,
який практично не залежить від величини суми. Це пояснюється тим,
що алгоритм перебирає лише фіксовану кількість номіналів монет.
Його часова складність становить O(n), де n — кількість номіналів.

Алгоритм динамічного програмування демонструє значно більший час виконання,
який зростає зі збільшенням суми. Це повʼязано з необхідністю обчислення
оптимального рішення для кожної проміжної суми від 0 до amount.
Часова складність алгоритму становить O(amount × n), а також
потребується додаткова памʼять для зберігання DP-таблиці.

Для даної системи монет [50, 25, 10, 5, 2, 1] жадібний алгоритм
є більш ефективним і доцільним для використання в касовій системі,
оскільки він забезпечує оптимальний результат при мінімальних витратах ресурсів.
